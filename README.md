# Hotel Revenue & Profitability Analysis 


Realizing that the hospitality industry—specifically hotels–relies heavily on data analysts to make data-driven decisions, I decided to conduct a case study analysis. This repository features a project that utilizes SQL to analyze a fictional hotel's revenue, profitability, and booking trends. The goal is to optimize pricing and operational efficiency based on a four-month dataset from 2024. This project uses a public dataset available on Kaggle.

## Project Overview: 
This project utilizes a public dataset available on Kaggle, and the Goal and Sample Questions are generated by AI.

1. Goal: Analyze hotel revenue, profitability, and booking trends to optimize pricing and operational efficiency.

Sample Questions to Answer:

- Revenue & Profitability: What are the patterns for Total Revenue, Room Revenue, Net Profit, and Profit Margin for this hotel in 2024?

- Occupancy & Room Performance: How are key metrics—such as Occupancy Rate, ADR, RevPAR, and Managed vs. All Guests—performing, and what do these figures signify?

- Booking Trends: What trends and patterns can be identified in New Bookings, No-Shows, Cancellations, and Channel Analysis?

- Guest Satisfaction: What are the key guest satisfaction metrics, such as average review scores, complaints, and compliments?

- Marketing Efficiency: How does marketing spend compare to revenue generated for this hotel?

2. Dataset
The data used for this project is available on [Kaggle](https://www.kaggle.com/datasets/omarsobhy14/hotel-revenue2024). You can query it directly in BigQuery or use your preferred SQL flavor (PostgreSQL, MySQL, etc.).

3. Methodology
- SQL query that extracts and transforms the data, including aggregate functions (SUM, AVG, and COUNT)

4. Tools Used
- SQL / BigQuery: For data extraction and aggregation.

- Looker: For data visualization.
